Code #1

Observations

All glyphs

Hint
Hope you know how to draw your glyphs
Solution

Reverse and split each glyph names
journey separate see potential thought destiny grow escape legacy rebel grid seek contemplate harm grid chaos conflict disorder grid data

For each glyph count the number of dots it crosses on the calibration grid. You can use the Glyphtionary to help you. Note that grid is not a glyph but the calibration grid and counts has 0.

You should get

7 7 2 5 9 6 3 5 9 6 0 5 8 6 0 8 6 8 0 5

We are now seeing a good decimal pattern. Reverse and make pairs

50 86 80 68 50 69 53 69 52 77

Convert Decimal to Ascii

2VPD2E5E4M

A few possible keywords can be used: calibration, grid, glyph, glyphs. All returned an invalid code.

Looking at glyphs more closely, the trap was with the Harm glyph which has a dot that needs to be counted twice.

This will then give
50 86 80 78 50 69 53 69 52 77

Convert Decimal to Ascii
2VPN2E5E4M

Again, a few possible keywords can be used: calibration, grid, glyph, glyphs

2VPN2GLYPHE5E4M

Code #2

Observations

17 characters which inclus 4 numbers. Almost in the passcode format.

Solution

This is used frequently by Niantic, simply reverse whatever is left and right of the middle letter, T in the case

6dpw5photinte2s5p

Code #3

Observations

Those # are there to throw you off, just a regular cipher used a lot

Hint
Those # really look like a grid right ?
Solution

Make a 4×4 grid

b o 3 7
# t t u
2 # y m
e 1 r t

Read down from 7

7umt3tyrot#1b#2e

Now why #1 and #2? They refer to the numbers in the code, 7 is the first number, 3 the second

7umt3tyrot7b3e
