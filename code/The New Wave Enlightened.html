Code #1

Observations

16 characters with enough numbers to make out a valid passcode

Solution

Make a 4×4 grid

7 p 8 a
9 e c g
t r o k
o 7 o 9

Read going down from the upper right A

agk98cooper779to

Code #2

Observations

All numbers, not in the decimal to Ascii range

Solution

Let’s start by making pairs to see if we can spot a pattern

36 32 29 00 08 21 24 38 38 09 08 07 45 22

Look at where the numbers should be in the passcode, they are all in the 00 to 09 range. For valid decimal to Ascii they would need to be in the 50 range. For the keyword, we have 21 24 38 38, the only keywords matching this pattern are Less and Loss. In decimal to Ascii S would need to be 83. Looks like we need to Rot +5 all the digits

81 87 74 55 53 76 79 83 83 54 53 52 90 77

Convert decimal to Ascii

QWJ75LOSS654ZM

Code #3

Observations

Again the right amount of numbers for a passcode but rectangle is not the cipher

Solution

After playing with different rectangle combinations nothing came up. A search for a keyword didn’t give anything either. I then tried a skip of 2 to get

uag65335szprrfktshnn

Split the string into two parts

uag65335sz
prrfktshnn

Last string sounds like perfection

uag65perfection335sz
