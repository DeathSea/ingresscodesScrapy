Inside the post “Not Currently Possible” on the Investigate Ingress blog, there are two codes hidden in the source and one displayed on the image itself:

[1] ad766f7bae8d7a889a879aab7aab9a
[2] sitnidenmoftimkpwnfip
[3] t3c4yiflb8vvp4

Code #1

Observations

Looks like hex, but on the other side of the ranges as what we need when pairing and converting them.

Hint

There is one cipher that we can always use when the ranges are the wrong way around!

Solution

Using the Hex version of Atbash (0123456789abcdef = fedcba9876543210), translate the given text.
528990845172857765786554855465
Pair them up
52 89 90 84 51 72 85 77 65 78 65 54 85 54 65
Convert from decimal to text for the final solve
4YZT3HUMANA6U6A

Code #2

Observations

We can spot some numbers in the code (si, fi, ni), so potentially we have everything there, just need to rearrange them.

Hint

Look for the keyword, then arrange them so to make it readable.

Solution

Form a grid of 3×7 with the text

s i t
n i d
e n m
o f t
i m k
p w n
f i p

Read each row normally, but starting at the last row (fip, then pwn, etc)
fippwnimkoftenmnidsit

Replace the written numbers with numbers for the final solve:
5ppw9mkoftenm9d6t

Code #3

Observations

Format is there, albeit in reverse order.
Seems easy enough, the normal solve steps should give us something.

Hint

When you see something that fits, try it on before exchanging it for something else.

Solution

Get the reverse out of the way first
4pvv8blfiy4c3t
Running the normal Atbash (with numbers Atbash this time, 123456789 = 987654321) against the code, we are left with this
6kee2yourb6x7g

From our list we see that your is not a known valid keyword.
After attempting to substitute it with anything that could be a valid keyword and getting no valid codes, we are left with just one option: Try it as is.
Putting the code into the scanner and getting a valid passcode response, we have found ourselfs a new keyword: your
