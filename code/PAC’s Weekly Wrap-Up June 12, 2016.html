Three codes were found in the PAC’s Weekly Wrap-Up: June 12, 2016 post on the Investigate Ingress blog.

[1] nnsmboa4fria48o5

[2] h3p3h05a1tneto401556

[3] fhirfhefcgficgfidhdiegjjafsdfdgh

Code #1

Observations

The four numbers are there with enough letter to have a valid passcode.

Solution

Make a 4×4 rectangle

n n s m
b o a 4
f r i a
4 8 o 5

Read up from bottom left 4

4fbn8ronoias5a4m

Ronioa is definitely not a valid keyword but Pronoia is. Just add the missing P.

4fbn8pronoias5a4m

Code #2

Observations

The first portion seems like the suffix, maybe that’s a good way to start.

Solution

Let’s reverse the string

655104otent1a50h3p3h

We know we have the suffix, h3p3h

Let’s look at the prefix 655104

Assuming that 6 and 4 are the numbers we are left with 5510 but we only need 3 letters to complete the prefix. 5510 can convert to roman numeral V V X.

We now have prefix and suffix

6vvx4 h3p3h

Now for the keyword, otent1a50. See some roman numerals in there? Convert the 1 to i and 50 to l

otential

From code #1 seems like keywords are missing the first letter, add the p to get potential

Put everything together

6vvx4potentialh3p3h

Code #3

Observations

All letters most of them between A to J but that R and S are out of bound !

Solution

Well letters from a to j could be 0 to 9 so let’s try this. Convert letters to numbers using a=0

5 7 8 17 5 7 4 5 2 6 5 8 2 6 5 8 3 7 3 8 4 6 9 9 0 5 18 3 5 3 6 7

Let make pairs to see if we get valid Ascii values

57 81 75 74 52 65 82 65 83 73 84 69 90 51 83 53 67

 Looks good, convert decimal to Ascii

9QKJ4ARASITEZ3S5C

Again first keyword letter is missing, add that P. Funny how all keywords started with P in this set.

9qkj4parasitez3s5c
