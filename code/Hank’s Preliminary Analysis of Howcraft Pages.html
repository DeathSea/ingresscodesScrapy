Three codes were hidden in the Hank’s Preliminary Analysis of Howcraft Pages post on the Investigate Ingress blog.

[1] 4pkn58tei46srxm

[2] 839kpu0va4622

[3] 0dw69030423sk

Code #1

Observations

15 characters with just the right amount of numbers for a passcode.

Solution

Make a 3×5 grid

4 p k
n 5 8
t e i
4 6 s
r x m

Read right from lower left r

rxm46stein584pk

Invalid, Stein is Stein Lightman

rxm46lightman584pk

Code #2

Observations

13 characters but there are way too many numbers.

Hint
You have a central point that you need to use to flip things around

Solution

13 characters so it cannot be a transposition cipher. With that many numbers, looking for a valid 3 letter keyword made it a bit more obvious. The keyword that fits is 802. Reverse the first and last groups of 6 characters, keeping the 0 in the middle.

upk93802264av

Code #3

Observations

You can sort of see the passcode format. You’ll need two different manipulations to get it.

Solution

Let’s split the prefix, keyword and suffix to see what we have

0dw69 030 423sk

To fix the prefix, convert that 0 from leetspeak to o

odw69 030 423sk

For the keyword part, using leetspeak would give oeo which is definitely not a keyword. Convert numbers to letters using a=0 gives ada

odw69ada423sk
